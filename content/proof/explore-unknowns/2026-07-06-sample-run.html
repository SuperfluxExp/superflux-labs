Task: build a Fable-style agent that takes a vague feature idea, explores unknowns before coding, and returns the one question most likely to change the implementation.

known knowns

  • Input is a vague feature idea. Output is one clarifying question, not a spec.
  • The workflow order is fixed: explore first, code never in this phase.
  • The question is ranked by leverage: how much the answer would change implementation.

known unknowns

  • What territory does the agent explore: idea only, or idea plus repo/docs?
  • Who answers: a human waiting synchronously, or an async review queue?
  • What is the exploration budget: seconds, or a multi-step pass?
  • What does "changes implementation" mean: data model, UX, or risk posture?

unknown knowns

  • Option A: one plain sentence, the question alone.
  • Option B: the question, why it is load-bearing, and what changes with each answer.
  • Option C: top three questions ranked, winner flagged.
  • A reaction like "B, but shorter" is the data we are after.

next question

When the agent asks its one question, who is on the other end: a human waiting synchronously, or an async review queue?

Synchronous means the run should finish in about 30 seconds and stay single-pass. Async permits a deeper sweep. This one answer decides runtime architecture, exploration budget, and whether a skip threshold is optional or essential.

decision-first plan

  1. Interaction contract: sync vs async answering.
  2. Input territory: idea only vs idea plus repo/docs.
  3. Output shape: pick A/B/C by reacting to samples.
  4. Skip condition: when asking is not worth the interruption.
  5. Then mechanics: exploration loop, ranking heuristic, answer-impact logging.
